
# Traffic Crash Analytics & Safety Intelligence Platform
# About the Project
This project leverages traffic crash data to extract actionable insights,
aiming to improve road safety, optimize emergency responses,
and support policy decisions. The platform demonstrates proficiency in SQL,
data analysis, and Python Streamlit integration.
# Business Use Cases
 * **Traffic Authorities:** Identifying high-risk streets and accident-prone zones.
 * **Insurance Companies:** Analyzing factors contributing to high-severity crashes.
 * **Emergency Services:** Evaluating response effectiveness using time-based data.
 * **Urban Planning:** Understanding the impact of road types and conditions.
 * **Research & Policy Making:** Studying crash trends and contributing causes.
# Key Skills
 * **SQL querying** and data analysis.
 * **Database management.**
 * **Python-SQL integration.**
 * **Streamlit table presentation.**
 * **Business insight generation.**
## Data Exploration & Analysis
The project includes in-depth analysis of traffic crashes, addressing complex queries such as:
 1. Finding dangerous weather and crash type combinations.
 2. Identifying top streets and high-risk locations.
 3. Analyzing crash trends by time, weather, and location.

    
 1. Transportation analytics is a rapidly evolving field that leverages data, statistical models, and machine learning to understand and improve mobility. By analyzing vehicle and transit patterns, this domain helps optimize traffic flow, reduce congestion, and enhance overall commuter experiences. It plays a critical role in urban planning and developing smart city infrastructures.
 2. The "Traffic Crash Analytics & Safety Intelligence Platform" is a comprehensive data analysis project designed to examine traffic accident datasets. It focuses on uncovering patterns, trends, and risk factors associated with traffic crashes. The project utilizes advanced SQL techniques and data visualization to provide actionable insights for road safety.
 3. The primary objective of this project is to analyze large-scale traffic crash data to identify high-risk areas and contributing factors to accidents. By uncovering these insights, the platform aims to support data-driven decision-making for traffic authorities and urban planners. Ultimately, it seeks to optimize emergency response and improve overall road safety measures.
 4. The ELT process began by extracting raw, pre-cleaned structured data from the provided data source. This data was then loaded into a SQL database table named CrashTable. Finally, transformations and analysis were performed directly within the database using advanced SQL queries, window functions, and aggregations to derive meaningful safety insights.

 5. During the exploratory data analysis using basic SQL queries, several key patterns were observed. Key findings indicate that certain weather and lighting conditions, as well as specific times of day, correlate with higher crash frequencies. Additionally, specific crash types, such as rear-end collisions, appear more frequently in the data.


 6. The project successfully analyzed large-scale traffic crash data, identifying high-risk locations and peak crash times. Key findings highlight the impact of weather and lighting conditions on accident frequencies, enabling better decision-making for traffic authorities to improve overall road safety measures.

 7. Business suggestion/solution: Based on the insights from the traffic data, authorities should implement targeted interventions such as optimizing traffic signals at high-risk intersections and improving road infrastructure in accident-prone zones to significantly reduce accident rates.
